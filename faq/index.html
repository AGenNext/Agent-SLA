Agent-SLA FAQ

What is Agent-SLA?

Agent-SLA is a DSL, SDK, API, MCP server, container package, and skill for AI-agent service level agreements.

Does Agent-SLA store runtime data?

No. Agent-Backend owns database and runtime state.

Which metrics are covered?

The implementation covers the AgentSLA Table 2 metric set, including precision, recall, TTFT, E2E, fairness, MCP, A2A, and oversight level.

Can I use Agent-SLA with MCP?

Yes. The MCP server exposes validation, drafting, evaluation, schema, quality model, and metric suggestion tools.

How are releases enforced?

Release workflows run unit tests, E2E smoke tests, security evaluation, CodeQL, and Docker builds.

Agent-SLA is maintained in AGenNext/Agent-SLA.
